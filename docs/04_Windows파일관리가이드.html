4.1.2 폴더명 매핑 (영문 ↔ 한글)

영문 폴더명 한글 설명 담당 AI 주요 파일
docs/ 문서 (개발계획서 등) 젠스파크 HTML 문서
specs/ 작업지시서 클로드 챗 Markdown 작업지시서
webapp/ 실제 개발 프로젝트 안티그래비티 Next.js 소스코드
reviews/ 리뷰 보고서 코워크 Markdown 리뷰 보고서
deliverables/ 최종 산출물 전체 배포 패키지, 문서
meetings/ 회의록 PM 회의록 문서
references/ 참고자료 전체 외부 자료, 스크린샷
⚠ 중요: 왜 영문 폴더명을 사용하나요?
  1. ✅ Git 호환성: 한글 경로는 일부 Git 클라이언트에서 문제 발생
  2. ✅ CI/CD 호환성: Linux 기반 빌드 서버에서 한글 경로 이슈
  3. ✅ URL 인코딩 방지: 배포 시 한글 경로는 %EC%xx 형태로 인코딩됨
  4. ✅ 국제 표준: 영문 폴더명은 국제 개발 표준
  5. ✅ 협업 편의성: 해외 개발자와 협업 시 문제 없음

4.2 파일 네이밍 규칙

기본 원칙:

  • 영문 또는 한글 사용 (특수문자 사용 금지: ?, *, :, <, >, |)
  • 공백 대신 언더스코어(_) 사용
  • 날짜 형식: YYYYMMDD (예: 20260307)
  • 버전 형식: vX.Y (예: v1.0, v2.3)

파일명 구조:

[문서종류]_[세부내용]_[버전]_[날짜].[확장자]

예시:

  • ✅ dev-plan_v1.0_20260308.html
  • ✅ task-002_license-apply_v1.0.md
  • ✅ review-001_initial-setup_20260309.md

잘못된 예시:

  • ❌ 개발 계획서 (최종).html (한글, 괄호 사용)
  • ❌ 작업지시서2.md (구조 불명확)
  • ❌ 리뷰보고서(수정본)!!!.md (특수문자 사용)

4.3 버전 관리 및 백업

4.3.1 버전 관리 방법

파일 기반 버전 관리 (Git 미사용 환경):

  • 원칙:
    • 절대 기존 파일을 덮어쓰지 않음
    • 새로운 버전은 새로운 파일로 저장
    • 변경 이력을 별도 문서로 관리

4.3.2 정기 백업 방법 (Windows 기준)

백업 전략: 3-2-1 규칙

  • 3: 3개의 복사본 유지 (원본 1 + 백업 2)
  • 2: 2가지 다른 저장 매체 사용 (로컬 HDD + 외장 HDD)
  • 1: 1개는 오프사이트(클라우드) 보관

백업 스케줄:

주기 방법 저장 위치
실시간 클라우드 동기화 OneDrive
매주 금요일 수동 ZIP 압축 외장 HDD
매월 말일 전체 아카이브 별도 아카이브 HDD
