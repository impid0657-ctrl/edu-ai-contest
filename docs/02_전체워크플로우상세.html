2.1 초기 개발 사이클 (1회차)

초기 개발 사이클

[1단계] 젠스파크 - 프로젝트 문서 제작

↓
  • 개발 계획서 작성 (HTML 형식)
  • 기능 명세서 작성
  • 사이트맵 및 IA 설계
  • Windows 폴더에 저장: 01_문서_젠스파크/
↓

[2단계] 클로드 챗 - 작업지시서 작성

↓
  • 젠스파크 문서 읽기
  • 개발 작업을 논리적 단위로 분해
  • 상세 작업지시서 작성 (Markdown)
  • Windows 폴더에 저장: 02_작업지시서_클로드/
↓

[3단계] 안티그래비티 - 코드 개발

↓
  • 작업지시서 기반 코딩
  • 기능 구현 및 테스트
  • 소스코드 작성
  • Windows 폴더에 저장: 03_개발결과_안티그래비티/
↓

[4단계] 코워크 - 작업 결과 리뷰

↓
  • 코드 품질 검증
  • 기능 동작 확인
  • 버그 및 개선사항 발견
  • 리뷰 보고서 작성
  • Windows 폴더에 저장: 04_리뷰보고서_코워크/
↓

[5단계] 클로드 챗 - 보고서 검토

↓
  • 코워크의 리뷰 보고서 분석
  • 수정 필요 사항 확인
  • 수정 작업지시서 작성
  • Windows 폴더에 저장: 02_작업지시서_클로드/
↓

[6단계] 안티그래비티 - 수정 작업

↓
  • 수정 지시서 기반 코드 수정
  • 버그 픽스 및 개선
  • 수정된 소스코드 저장
↓

[반복] 4~6단계 반복 (이슈 없을 때까지)

↓

[완료] 최종 산출물 폴더로 이동

주요 체크포인트:

  • ✅ 각 단계 완료 시 담당자 확인
  • ✅ 산출물이 지정된 폴더에 정확히 저장되었는지 확인
  • ✅ 파일명이 네이밍 규칙을 준수하는지 확인

2.2 반복 개선 사이클 (2회차~)

사이클 시작 조건:

  • 코워크의 리뷰에서 문제가 발견된 경우
  • 새로운 요구사항이 추가된 경우
  • 기능 개선이 필요한 경우

반복 사이클:

반복 개선 사이클

[시작] 이슈 발견 (코워크 리뷰)

↓

[1] 클로드 챗 - 리뷰 보고서 분석

  • 문제 원인 파악
  • 해결 방법 설계
  • 수정 작업지시서 작성
↓

[2] 안티그래비티 - 수정 작업

  • 지시서 기반 코드 수정
  • 테스트 수행
  • 수정 완료 보고
↓

[3] 코워크 - 재검증

  • 수정 사항 확인
  • 새로운 이슈 체크
  • 리뷰 보고서 갱신
↓

[판단] 문제 해결 여부?

  • Yes → [완료] 다음 작업으로 이동
  • No → [1]로 돌아가서 재작업

반복 사이클 종료 조건:

  • ✅ 코워크의 리뷰에서 Critical/Major 이슈 0건
  • ✅ 모든 기능이 정상 동작
  • ✅ 코드 품질 기준 충족
