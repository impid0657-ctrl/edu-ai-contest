8.1 프로젝트 개요

본 프로젝트는 제8회 교육 공공데이터 AI활용대회의 온라인 접수 및 운영을 위한 웹 플랫폼을 구축하는 것입니다. 참가자의 접근성을 높이고, 대용량 파일의 안정적인 접수, AI 기술을 활용한 사용자 지원 및 심사 보조 기능을 제공하는 것을 목표로 합니다.

8.5 데이터베이스 스키마 설계

Supabase PostgreSQL 데이터베이스의 주요 테이블 DDL입니다.

1. 사용자 테이블 (users)
create table public.users (
  id uuid references auth.users on delete cascade not null primary key,
  email text,
  name text,
  phone text,
  role text default 'user', -- 'user', 'admin'
  created_at timestamp with time zone default timezone('utc'::text, now())
);
2. AI 이용권 신청 테이블 (license_applications)
create table public.license_applications (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users(id),
  team_name text,
  category text not null,
  school_name text,
  status text default 'pending', -- 'pending', 'approved', 'rejected'
  created_at timestamp with time zone default timezone('utc'::text, now())
);
3. 작품 접수 테이블 (submissions)
create table public.submissions (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users(id), -- 비회원인 경우 null 가능
  submission_no text unique not null, -- 접수번호
  title text not null,
  description text,
  category text not null,
  password_hash text, -- 비회원 접수용 비밀번호 (bcrypt Hashed) [v1.2 수정]
  contact_email text, -- 비회원 연락처
  status text default 'submitted',
  ai_review_status text default 'pending', -- [v1.2 추가] AI 심사 상태
  created_at timestamp with time zone default timezone('utc'::text, now())
);
4. 제출 파일 관리 테이블 (submission_files)
create table public.submission_files (
  id uuid default gen_random_uuid() primary key,
  submission_id uuid references public.submissions(id) on delete cascade,
  file_name text not null,
  file_path text not null, -- Storage 경로
  file_size bigint,
  mime_type text,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
5. AI 심사 결과 테이블 (ai_review_results)
create table public.ai_review_results (
  id uuid default gen_random_uuid() primary key,
  submission_id uuid references public.submissions(id) on delete cascade,
  score jsonb, -- { "creativity": 80, "tech": 90 }
  summary text,
  feedback text,
  processed_at timestamp with time zone default timezone('utc'::text, now())
);
6. 게시판 테이블 (posts)
create table public.posts (
  id uuid default gen_random_uuid() primary key,
  type text not null, -- 'notice', 'faq', 'qna', 'gallery'
  title text not null,
  content text,
  author_id uuid references public.users(id),
  is_secret boolean default false,
  view_count integer default 0,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
7. 챗봇 로그 테이블 (chatbot_logs)
create table public.chatbot_logs (
  id uuid default gen_random_uuid() primary key,
  user_query text,
  ai_response text,
  is_blocked boolean default false,
  tokens_used integer,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
8. 대회 요강 벡터 테이블 (contest_documents)
create table public.contest_documents (
  id uuid default gen_random_uuid() primary key,
  content text,

  metadata jsonb

9. 관리자 페이지 트리 (pages)
create table public.pages (
  id uuid default gen_random_uuid() primary key,
  parent_id uuid references public.pages(id),
  title text not null,
  path text,
  order_index integer,
  is_visible boolean default true

8.6 주요 기능 상세 명세

8.6.1 이용권 신청 프로세스

  1. 로그인/인증: 카카오/네이버/학교 이메일 인증
  2. 신청서 작성: 학교명, 부문 등 입력
  3. DB 저장: 초기 상태는 pending (대기중)
  4. 관리자 승인: 관리자가 검수 후 일괄 승인 → approved 상태 변경
  5. 에듀핏 연동: 승인된 사용자만 CSV 추출하여 에듀핏 전달

8.6.2 참가작 접수 프로세스

  • 파일 업로드:
    • 회원: Supabase Storage 직접 업로드 (RLS 적용)
    • 비회원: /api/guest/upload API 경유 업로드 (Service Role Key 사용)
  • AI 심사: 자동 웹훅 방식 폐기. 관리자가 대시보드에서 수동 트리거 버튼 클릭 → 백그라운드 큐 처리

8.6.3 AI 챗봇 시스템 (RAG 방식)

  • 벡터 DB: 대회 요강 데이터를 청킹하여 contest_documents에 저장
  • 검색 로직: 사용자 질문 벡터화 → 코사인 유사도 검색 → LLM 프롬프트 주입
  • 설정 관리: chatbot_settings 테이블을 통해 모델 및 프롬프트 동적 관리

8.6.4 게시판 시스템

  • 공지사항: 관리자만 작성, 전체 열람
  • 비밀게시판: 작성자와 관리자만 열람 (RLS 적용)

8.8 인증 시스템 구현 상세

  • Supabase Auth: 카카오, 네이버 OAuth Provider 설정
  • 학교 이메일 인증: 커스텀 OTP (Node.js + Resend API)
  • 비회원 인증: bcrypt 해시 비밀번호 + HttpOnly Cookie/JWT 커스텀 세션 구현 (Supabase Auth 미사용)

8.9 파일 업로드 및 스토리지 관리

  • 저장소: Supabase Storage (Pro 플랜 권장)
  • RLS 정책: 회원은 auth.uid() 기반, 비회원은 API Proxy 사용
  • 파일 관리: file_upload_settings 테이블에서 허용 확장자 및 크기 관리

8.10 AI 챗봇 설정 코드

import { OpenAIStream, StreamingTextResponse } from 'ai';
import { Configuration, OpenAIApi } from 'openai-edge';
import { createClient } from '@supabase/supabase-js';

export async function POST(req: Request) {
  const { messages } = await req.json();
  const lastMessage = messages[messages.length - 1];

  // 1. Embed user query
  const embeddingRes = await openai.createEmbedding({
    model: 'text-embedding-ada-002',
    input: lastMessage.content,
  });
  const embedding = embeddingRes.data.data[0].embedding;

  // 2. Search Vector DB
  const { data: documents } = await supabase.rpc('match_documents', {
    query_embedding: embedding,
    match_threshold: 0.78,
    match_count: 5,
  });

  // 3. Construct System Prompt
  const context = documents.map(doc => doc.content).join('\n');
  const systemPrompt = `You are a helpful assistant for the AI competition.
  Use the following context to answer:\n${context}`;

  // 4. Generate Response
  const response = await openai.createChatCompletion({
    model: 'gpt-4',
    stream: true,
    messages: [{ role: 'system', content: systemPrompt }, ...messages],
  });

  return new StreamingTextResponse(OpenAIStream(response));
}

8.11 보안 정책

  • RLS (Row Level Security): DB 레벨에서 사용자 본인 데이터만 접근 강제
  • API 보안: Service Role Key는 오직 서버 사이드 API Route에서만 사용 (브라우저 노출 절대 금지)
  • Rate Limiting: Upstash Redis 등을 활용하여 DDoS 방지
