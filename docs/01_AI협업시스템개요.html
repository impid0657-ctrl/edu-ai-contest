1.1 시스템 구성 목적

본 프로젝트는 제8회 교육 공공데이터 AI활용대회 홈페이지 개발을 위해 4개의 전문화된 AI 도구를 협업 시스템으로 구성하여 운영합니다.

핵심 목표:

  • 문서화 자동화 및 품질 향상: 일관된 형식의 전문적인 개발 문서 자동 생성
  • 작업 지시의 명확성 확보: 모호함을 제거한 구체적인 작업 지시
  • 개발 작업의 정확도 제고: 표준화된 프로세스를 통한 오류 최소화
  • 체계적인 품질 검증 프로세스 확립: 단계별 검증을 통한 품질 보장
  • 반복 개선 사이클의 효율화: 신속한 피드백 반영 및 수정

기대 효과:

  • 개발 시간 단축 (약 30% 예상)
  • 의사소통 오류 최소화
  • 작업 이력 완벽 추적
  • 품질 일관성 유지

1.4.1 핵심 기능 3가지

🔹 기능 1: AI 이용권 신청 + 작품 접수 시스템

목적: 참가자가 AI 이용권(에듀핏 라이선스)을 신청하고, 작품을 온라인으로 제출합니다.

[AI 이용권 신청]

  • 간편 인증 후 신청 (다음 중 선택):
    • ✅ 카카오 (보호자/성인 인증)
    • ✅ 네이버 (보호자/성인 인증)
    • ✅ 학교 이메일 (학생 본인 인증 - *.ac.kr, *.hs.kr, *.ms.kr, *.es.kr 등)
    • ⏸ PASS/Payco/Toss (향후 추가 예정 - 승인 절차 소요 기간 고려)
  • 1인 1회 신청 제한
  • 승인 및 마감 정책 (최종):
    • 모든 신규 신청자는 pending(대기중) 상태로 저장됩니다.
    • 관리자가 대시보드에서 검수 후 일괄 승인(Bulk Approve) 버튼을 눌러야 승인 처리됩니다.
    • 500명 마감 기준은 '전체 신청자'가 아닌, '최종 승인(approved)된 인원 수'를 기준으로 합니다.

[작품 접수]

  • 회원/비회원 모두 접수 가능 (비회원은 이메일+비밀번호)
  • 3개 부문 (초등 영상, 중·고등 AI분석, 일반 AI서비스)
  • 파일 업로드 (최대 500MB, 팀당)
  • 접수번호 자동 발급 및 이메일 발송
  • 본인 확인 후 수정 가능 (마감 전까지)
  • 기술: Supabase Database + Supabase Storage + 다중 인증 제공자

🔹 기능 2: AI 챗봇 (대회 요강 자동 QA)

목적: 참가자가 대회 요강 정보를 24시간 자동으로 질의응답할 수 있도록 지원합니다.

  • 주요 기능:
    • 웹사이트 우측 하단에 플로팅 버튼 (모든 페이지 접근 가능)
    • 대회 요강 문서를 벡터 DB(pgvector)에 저장 → RAG 방식으로 답변
    • 답변 범위 제한 (자격, 일정, 접수, 시상, FAQ만 답변)
    • 범위 외 질문 차단: "대회 요강 관련 질문만 답변 가능합니다."
    • AI 제공자 선택 가능 (OpenAI, Anthropic, Google, Cohere, Perplexity)
    • 비용 추적 (토큰 사용량, 월별 비용 집계)
    • 질문/답변 로그 저장 (관리자 대시보드에서 조회)
  • 기술: Next.js API Routes + Supabase pgvector + 다중 AI 제공자 지원

🔹 기능 3: 관리자 대시보드

목적: 신청·접수 데이터 관리, 심사, 설정

  • 주요 기능:
    • 이용권 신청 관리: 신청자 목록, 승인/반려, 에듀핏 CSV 다운로드
    • 작품 접수 관리: 접수 목록, AI 심사 결과 확인, 파일 다운로드
    • 학생 인증 관리: 이메일 OTP 자동 승인, 학생증 업로드 수동 승인
    • 게시판 관리: 공지사항 작성, 모든 게시글 수정/삭제
    • 챗봇 설정: AI 제공자 선택, 답변 범위 설정, 예시 질문 편집, 비용 추적
    • 인증 제공자 설정: 카카오/네이버/학교 이메일 활성화
    • 페이지 권한 설정: 공개/회원/관리자 접근 제어
    • 통계 대시보드: 실시간 접수 현황, 부문별 통계, 지역별 분포
  • 접근 권한: 관리자 전용 (role='admin')

1.4.2 에듀핏 연동 방식

단계 내용
1. 데이터 수집 이용권 신청 데이터는 우리 DB (Supabase)에 저장 (기본 상태: pending)
2. 데이터 확인 관리자 대시보드에서 이용권 신청자 목록 조회 (실시간)
3. 검수 및 승인 관리자가 신청자를 선택하여 일괄 승인(Bulk Approve) 실행.
시스템은 승인된 인원(approved)이 500명을 초과하지 않도록 검증 후 상태를 업데이트합니다.
4. 데이터 전달 방법 1 (권장): 관리자 대시보드에서 '승인된 사용자(approved)'만 필터링하여 CSV/엑셀 다운로드 → 이메일 또는 공유 드라이브로 에듀핏에 전달
방법 2 (협의 필요): 에듀핏 API가 제공되는 경우 자동 연동
5. 에듀핏 처리 에듀핏 측에서 라이선스 발급 및 이메일 발송 (사용 기간: 4/1~5/31)

중요 사항:

  • ✅ 이용권 신청 데이터는 우리 시스템(Supabase)에서 관리
  • ✅ 에듀핏은 단순히 라이선스 발급 전문 업체 역할
  • ✅ 데이터 전달은 일괄 처리 (Batch) 방식
  • ✅ 관리자가 전달 시점을 직접 제어
