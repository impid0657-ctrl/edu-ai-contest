9.1 용어 정의

용어 설명
RAG Retrieval-Augmented Generation. LLM이 외부 지식 베이스(벡터 DB)에서 관련 정보를 검색하여 답변을 생성하는 기술.
pgvector PostgreSQL에서 고차원 벡터 데이터를 저장하고 유사도 검색을 수행할 수 있게 해주는 확장 모듈.
RLS Row Level Security. 데이터베이스의 각 행(Row) 단위로 접근 제어 규칙을 적용하는 보안 기능.
Supabase Firebase의 오픈 소스 대안으로, PostgreSQL, 인증, 스토리지 등을 제공하는 BaaS 플랫폼.
Edge Functions 사용자와 가까운 위치(CDN Edge)에서 실행되는 서버리스 함수. 지연 시간이 매우 짧음.

9.2 자주 묻는 질문 (FAQ)

개발 관련

  • Q: 로컬 환경에서 Supabase는 어떻게 실행하나요?
    A: npx supabase start 명령어를 통해 Docker 컨테이너 기반 로컬 Supabase 인스턴스를 실행할 수 있습니다.
  • Q: 대용량 파일 업로드가 실패합니다.
    A: Vercel 함수 용량 제한 때문일 수 있습니다. 8.9절의 Direct Upload 방식을 확인하세요.

운영 관련

  • Q: 접수 기간 연장은 어떻게 하나요?
    A: 관리자 대시보드 > 설정 메뉴에서 접수 종료일을 수정하면 즉시 반영됩니다.
  • Q: 챗봇 답변이 이상합니다.
    A: 벡터 DB의 문서 데이터를 최신화하거나, 시스템 프롬프트를 수정하여 답변 범위를 조정해야 합니다.

9.4 인증 제공자 연동 가이드

9.4.1 카카오 OAuth 연동 가이드

  1. 애플리케이션 등록: Kakao Developers 접속 > 내 애플리케이션 > 애플리케이션 추가.
  2. 플랫폼 설정: Web 플랫폼 등록 > 사이트 도메인 입력 (예: https://edu-ai.kr).
  3. Redirect URI 설정: https://[project-id].supabase.co/auth/v1/callback 등록.
  4. Client Secret 발급: 보안 메뉴에서 Client Secret 생성 및 활성화.
  5. Supabase 설정: Auth > Providers > Kakao 활성화 > REST API Key 및 Client Secret 입력.

9.4.2 네이버 OAuth 연동 가이드

  1. 애플리케이션 등록: Naver Developers 접속 > Application > 애플리케이션 등록.
  2. API 설정: '네이버 로그인' 선택 > 제공 정보(이름, 이메일 등) 필수 체크.
  3. Callback URL 설정: https://[project-id].supabase.co/auth/v1/callback 등록.
  4. Supabase 설정: Auth > Providers > Naver 활성화 > Client ID 및 Client Secret 입력.

9.4.3 학교 이메일 OTP 인증 구현 가이드

OAuth를 사용할 수 없는 학생들을 위한 이메일 기반 인증입니다.

1. 도메인 검증 로직 (Regex)

/^[a-zA-Z0-9._%+-]+@([a-zA-Z0-9-]+\.)?(ac\.kr|hs\.kr|ms\.kr|es\.kr|go\.kr)$/

2. 프로세스

  1. 사용자가 이메일 입력 시 위 정규식으로 학교/기관 도메인인지 1차 검증.
  2. 통과 시 6자리 랜덤 숫자 생성하여 DB(Redis)에 Key:Email, Value:OTP, TTL:5분으로 저장.
  3. 해당 이메일로 인증번호 발송.
  4. 사용자가 입력한 번호와 DB 저장 값 대조 후 일치 시 인증 완료 토큰 발행.

3. 구현 코드 예시

// lib/auth/school-email-otp.ts 구현 예시
import { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendOTP(email: string) {
  // 1. 학교 도메인 검증
  const schoolDomainRegex = /^[a-zA-Z0-9._%+-]+@([a-zA-Z0-9-]+\.)?(ac\.kr|hs\.kr|ms\.kr|es\.kr|go\.kr)$/;

  if (!schoolDomainRegex.test(email)) {
    throw new Error('학교 이메일이 아닙니다.');
  }

  // 2. 6자리 랜덤 OTP 생성
  const otp = Math.floor(100000 + Math.random() * 900000).toString();

  // 3. Redis에 저장 (TTL: 5분) - 실제 구현 시 Redis 클라이언트 사용
  // await redis.setex(`otp:${email}`, 300, otp);

  // 4. 이메일 발송
  await resend.emails.send({
    from: '[email protected]',
    to: email,
    subject: '[AI활용대회] 인증번호를 확인해주세요',
    html: `인증번호: ${otp} (5분간 유효)`
  });

  return { success: true };
}

export async function verifyOTP(email: string, inputOTP: string) {
  // const storedOTP = await redis.get(`otp:${email}`);
  const storedOTP = "123456"; // 예시

  if (storedOTP === inputOTP) {
    // await redis.del(`otp:${email}`); // OTP 사용 후 삭제
    return { verified: true };
  }

  return { verified: false, error: '인증번호가 일치하지 않습니다.' };
}
