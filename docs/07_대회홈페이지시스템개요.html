7.3 사용자 흐름 (User Flow)

🔵 흐름 1: AI 이용권 신청 (필수 인증)

  1. 사용자가 "AI 이용권 신청" 페이지 접속
  2. "로그인 후 신청하기" 버튼 클릭
  3. 간편 인증 선택 (카카오/네이버/PASS/페이코/토스 중 관리자가 활성화한 제공자)
  4. 인증 성공 → Supabase Auth에 자동 회원 등록
  5. 본인 정보(이름, 이메일, 연락처) 자동 입력
  6. 신청 정보 입력 (학년, 학교명 등)
  7. 신청 완료 → DB에 저장 (pending 상태, 관리자 승인 대기)
  8. 관리자 검수 및 일괄 승인 → 에듀핏 라이선스 발급

💡 왜 인증이 필요한가?

  • 무분별한 중복 신청 방지
  • 신청자 신원 확인
  • 에듀핏 라이선스 발급 시 본인 확인 필요

🟢 흐름 2: 작품 접수 (선택 인증)

Option A: 인증 접수 (간편 입력)

  1. 사용자가 "작품 접수" 페이지 접속
  2. "로그인 후 접수" 버튼 클릭 → 간편 인증
  3. 본인 정보(이름, 이메일, 연락처) 자동 입력
  4. 작품 파일 업로드 (압축 파일, 최대 500MB)
  5. 접수 완료 → 접수번호 자동 발급 및 이메일 발송
  6. 마이페이지에서 접수 내역 확인 가능
  7. 수정 필요 시 → 로그인 후 수정 (마감 전까지)

Option B: 비인증 접수 (이메일/비밀번호)

  1. 사용자가 "작품 접수" 페이지 접속
  2. "이메일로 접수" 버튼 클릭
  3. 이메일, 연락처, 비밀번호(4자리 이상) 직접 입력
  4. 작품 파일 업로드 (서버 API 경유 업로드)
  5. 접수 완료 → 접수번호 이메일 발송
  6. 수정 필요 시 → 커스텀 JWT/HttpOnly 쿠키 세션 사용 (접수번호 + 이메일 + 비밀번호 입력으로 본인 확인 후 수정)

💡 왜 선택 인증인가?

  • 참가 접근성 향상 (간편 인증 없이도 참가 가능)
  • 회원은 자동 입력으로 편의성 증대
  • 비회원은 이메일로 접수번호 확인 및 수정 가능

🟡 흐름 3: 게시판 (비회원 허용)

비회원 글 작성

  1. 게시판 → "글쓰기" 버튼 클릭
  2. 작성자 이름, 비밀번호(4자리 이상) 입력
  3. 글 내용 작성 후 등록
  4. 수정/삭제 시 → 비밀번호 입력으로 본인 확인

회원 글 작성

  1. 게시판 → "로그인 후 글쓰기" 버튼 클릭
  2. 간편 인증 완료
  3. 작성자 이름 자동 입력, 비밀번호 불필요
  4. 글 내용 작성 후 등록
  5. 수정/삭제 시 → JWT 토큰 인증으로 본인 확인 (비밀번호 불필요)

🔴 흐름 4: 관리자 대시보드

  1. 관리자 계정으로 로그인 (role='admin')
  2. 관리자 대시보드 접속
  3. 주요 기능:
    • 이용권 신청 목록 (검수 후 일괄 승인/반려)
    • 작품 접수 목록 (AI 심사 수동 트리거 및 결과 확인)
    • 학생 인증 승인/반려
    • 게시판 관리 (모든 글 수정/삭제)
    • 페이지 권한 설정 (공개/회원/관리자)
    • 인증 제공자 설정 (카카오/네이버/학교 이메일 활성화)
    • 챗봇 설정 (AI 제공자, 답변 범위, 예시 질문)
    • 파일 업로드 설정 (최대 용량, 허용 확장자)

7.4 인증 시스템 정책

7.4.1 현재 지원 인증 제공자 (즉시 사용 가능)

인증 제공자 특징 및 장점
✅ 카카오 (Kakao) - 국내 1위 메신저 앱, 대부분의 사용자가 보유
- 빠른 연동 (Supabase Auth 기본 지원)
- 무료 사용 가능
- 보호자/성인 인증으로 사용
✅ 네이버 (Naver) - 국내 2위 포털, 카카오 없는 사용자 대체
- OAuth 2.0 표준 지원
- 무료 사용 가능
- 보호자/성인 인증으로 사용
✅ 학교 이메일 OTP - 학생 본인 인증 전용 (*.ac.kr, *.hs.kr, *.ms.kr, *.es.kr 등)
- OTP 6자리 발송 → 90% 자동 승인
- 인증 실패 시 학생증 업로드 → 관리자 수동 승인
- 커스텀 구현 (Supabase Edge Function + Resend 이메일 API)

인증 제공자 특징 승인 소요 시간

✅ OAuth 2.0 표준 지원
✅ 무료 사용 가능		 1-3 영업일

7.4.3 학생 인증 흐름

📌 학생 AI 이용권 신청 시 인증 방법: 다음 두 가지 방법 중 하나를 선택하여 인증할 수 있습니다.

인증 방법 절차 설명
방법 A
보호자 인증 		① 보호자가 카카오 또는 네이버로 간편 인증
② 이용권 신청 폼 작성
③ 관리자 승인 후 발급		 ✅ 초등학생 또는 스마트폰이 없는 학생 권장
✅ 보호자의 카카오/네이버 계정 사용
방법 B
학생 본인 이메일 인증 		① 학교 이메일 주소 입력 (*.ac.kr, *.hs.kr 등)
② OTP 6자리 이메일로 발송
③ OTP 입력 → 90% 자동 승인
④ (실패 시) 학생증 업로드 → 관리자 수동 승인 		✅ 중·고등·대학생 권장
✅ 학교 이메일이 있는 경우
⚠ OTP 인증 실패 시 학생증 업로드 필수

🚨 [v1.2 추가] 비회원 인증 아키텍처 (최종 확정)

Supabase Auth를 통한 비회원(접수번호+비밀번호) 로그인은 지원되지 않습니다. 따라서 아래와 같이 독자적인 커스텀 인증 세션을 구현해야 합니다.

  • 1. 비밀번호 저장: 접수 시 입력한 비밀번호를 bcrypt(saltRounds 12 이상)로 해싱하여 DB에 저장합니다.
  • 2. 로그인 요청: POST /api/guest/login으로 접수번호와 비밀번호를 전송합니다.
  • 3. 검증 및 세션 발급: 서버에서 bcrypt로 비밀번호를 대조한 후, HttpOnly 쿠키 또는 커스텀 JWT를 발급하여 세션을 유지합니다.
  • 4. 보안: 발급된 토큰은 비회원 파일 업로드 등 후속 작업의 권한 검증에 사용됩니다.

7.5 데이터 흐름 및 보안

🔒 보안 정책

  • 인증: Supabase Auth + JWT 토큰 (Access Token 1시간, Refresh Token 7일)
  • 세션 관리: HttpOnly 쿠키 + CSRF 토큰
  • 데이터베이스: Supabase RLS (Row Level Security) - 본인 데이터만 조회 가능
  • 파일 업로드: Supabase Storage 서명된 URL (회원), API 경유 Service Role 업로드 (비회원), 바이러스 검사 (선택)
  • API 호출: JWT 검증, 관리자 권한 확인, Rate Limiting

7.6 시스템 특장점

  • ✅ 높은 접근성: 비회원도 작품 접수 및 게시판 작성 가능
  • ✅ 편리한 인증: 5개 간편 인증 제공자 중 선택 (회원가입 불필요)
  • ✅ 본인 확인: 이메일/비밀번호 또는 간편 인증으로 본인 확인
  • ✅ AI 기반 QA: 챗봇이 대회 요강 범위 질문에 자동 답변 (비용 추적)
  • ✅ AI 심사: 제출 파일 분석 및 점수 산출 (관리자가 수동 트리거 + 백그라운드 큐 처리, 타임아웃 없음)
  • ✅ 유연한 관리: 관리자가 인증 제공자, 챗봇, 파일 업로드 설정 가능
  • ✅ 선착순 제한: AI 이용권 500명 승인 기준 관리 (검수 후 승인)
  • ✅ 실시간 대시보드: 관리자가 실시간으로 신청/접수 현황 확인
