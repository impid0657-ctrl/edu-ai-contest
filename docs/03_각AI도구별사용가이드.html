3.2 클로드 챗 작업지시서 작성 가이드

3.2.1 주요 역할 및 책임

  • 젠스파크 문서를 읽고 개발 작업으로 변환
  • 명확하고 실행 가능한 작업지시서 작성
  • 코워크의 리뷰 보고서를 분석하여 수정 지시서 작성
  • 전체 프로젝트의 기술적 중재자 역할

3.2.2 좋은 작업지시서의 조건

SMART 원칙 준수:

  • Specific (구체적): "로그인 기능 구현" ❌ → "카카오 간편인증 API를 사용한 로그인 기능 구현" ✅
  • Measurable (측정 가능): "빠르게 동작" ❌ → "3초 이내 응답" ✅
  • Achievable (달성 가능): 현실적인 범위와 기간 설정
  • Relevant (관련성): 프로젝트 목표와 연결
  • Time-bound (기한 명시): "언젠가" ❌ → "3일 내" ✅

3.2.4 수정 작업지시서 작성 방법

코워크의 리뷰 보고서 분석:

  1. 발견된 이슈 목록 확인
  2. 이슈별 우선순위 평가 (Critical → Major → Minor)
  3. 수정 방법 설계
  4. 수정 작업지시서 작성

수정 지시서 예시:

# 수정지시서 FIX-001 - 이용권 신청 폼 유효성 검증 오류 수정

1. 이슈 요약
- 이슈 ID: BUG-023 (코워크 리뷰보고서 참조)
- 심각도: Critical
- 발견일: 2026-03-15
- 발견자: Cowork

2. 문제 상황
이용권 신청 시 휴대폰번호 필드에 영문자 입력이 가능하여 잘못된 데이터가 DB에 저장되는 문제 발생.

3. 원인 분석
input 태그의 type이 "text"로 설정되어 있으며, 클라이언트 사이드 유효성 검증이 누락됨.

4. 수정 지시사항
### 4.1 HTML 수정
- input type을 "tel"로 변경
- pattern 속성 추가: pattern="[0-9]{10,11}"

### 4.2 JavaScript 수정
- 정규식 기반 유효성 검증 추가
- 에러 메시지 표시 로직 구현

3.3 안티그래비티 개발 작업 가이드

3.3.1 주요 역할 및 책임

  • 클로드 챗의 작업지시서를 받아 실제 코드 구현
  • 기능 개발 및 단위 테스트 수행
  • 소스코드 및 관련 파일 작성
  • 간단한 기술 문서 작성

3.3.2 작업지시서 수신 및 분석

작업 시작 전 체크리스트:

  • [ ] 작업지시서를 끝까지 정독
  • [ ] 불명확한 부분이 있으면 클로드 챗에게 질문
  • [ ] 필요한 입력 데이터(젠스파크 문서, 기존 코드 등)를 모두 확보
  • [ ] 개발 환경이 준비되었는지 확인
  • [ ] 예상 소요 시간 산정

3.3.3 개발 작업 수행

개발 순서:

  1. 환경 설정: 필요한 라이브러리, 도구 설치
  2. 코드 작성: 작업지시서의 요구사항 구현
  3. 자체 테스트: 기본적인 동작 확인
  4. 주석 및 문서화: 코드에 주석 추가, README 작성
  5. 파일 정리: 불필요한 파일 삭제, 폴더 구조 정리

3.3.4 산출물 작성 및 저장

필수 산출물:

  1. 소스코드 (HTML, CSS, JS, Python 등)
  2. README.md (코드 설명, 실행 방법)
  3. 테스트 결과 스크린샷 (선택)
  4. 설치 파일 또는 빌드 결과물

작업 구조 예시:

webapp/                 # 메인 Next.js 프로젝트
├── app/
├── components/
├── lib/
└── README.md

참고: 안티그래비티는 webapp/ 폴더에서 직접 개발하며, 완료 보고서는 별도로 작성합니다.

3.4.1 주요 역할 및 책임

3.4.2 리뷰 체크리스트

1. 기능 검증 (Functional Review)

  • [ ] 기능이 정상적으로 동작하는가?
  • [ ] 예외 상황(에러)을 적절히 처리하는가?

2. 코드 품질 (Code Quality)

  • [ ] 코드가 읽기 쉽고 이해하기 쉬운가?
  • [ ] 적절한 주석이 달려 있는가?
  • [ ] 코딩 표준을 준수하는가?

3. 보안 (Security)

  • [ ] SQL Injection / XSS 취약점은 없는가?
  • [ ] 중요 정보(API Key)가 노출되지 않았는가?

3.4.3 이슈 우선순위 분류

우선순위 설명 조치
Critical 시스템 작동 불가, 보안 취약점 즉시 수정 필수

Minor 경미한 버그, UI 개선 다음 버전에서 수정
