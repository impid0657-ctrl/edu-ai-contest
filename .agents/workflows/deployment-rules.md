# 배포 규칙

## 1. 배포는 성민의 명시적 지시가 있어야만 수행

- `git push origin main`은 **성민이 "배포해"라고 말한 경우에만** 실행한다.
- 코드 수정, 빌드 검증까지는 자율 수행 가능하지만, **push는 절대 자율 수행 금지**.
- 빌드 검증 완료 후 성민에게 변경 내역을 보고하고, 배포 지시를 기다린다.

## 2. 프로덕션에 영향 가능한 작업은 사전 보고

아래 작업은 실행 전 반드시 성민에게 **위험 내용과 영향 범위를 보고**한다:

- 공개 페이지(public) 레이아웃, 헤더, 푸터 변경
- 메뉴 노출/공개여부 관련 로직 변경
- API 엔드포인트의 인증/권한 로직 변경
- DB 마이그레이션 (스키마 변경)
- 환경변수(.env) 변경
- 외부 서비스 연동 변경 (Supabase, Vercel, OpenRouter 등)
- CSS/JS 로드 순서 변경

## 3. 로컬 검증 우선

- 모든 변경은 `localhost:3000`에서 먼저 확인한다.
- 로컬에서 문제없음을 확인한 후에만 배포 가능 상태로 보고한다.

## 4. 작업 흐름

```
코드 수정 → 로컬 빌드 검증(next build) → 로컬 브라우저 확인(localhost:3000) → 성민에게 보고 → 성민 "배포해" → git push
```
